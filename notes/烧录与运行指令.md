# 烧录与运行指令

本文档根据当前工程的 `platformio.ini` 和 `pixi.toml` 整理。工程里共有 4 个 PlatformIO 固件环境：

| 单片机角色 | PlatformIO 环境 | 源码目录 | 默认烧录端口 | 说明 |
| --- | --- | --- | --- | --- |
| Master 主机 | `master` | `src/master/` | `/dev/cu.usbmodem1101` | ADS1298 采集、ESP-NOW 时间基准和 USB CDC 数据汇聚 |
| Slave 01 从机 | `slave-01` | `src/slave-01/` | `/dev/cu.usbmodem101` | MPU6500 陀螺仪采集节点 |
| Slave 02 从机 | `slave-02` | `src/slave-02/` | `/dev/cu.usbmodem101` | MPU6500 加速度采集节点 |
| Motor 电机节点 | `motor` | `src/motor/` | `/dev/cu.usbmodem1101` | BLE 电机控制节点 |

## 推荐烧录方式

项目已经在 `pixi.toml` 里封装好了烧录任务，推荐优先使用下面这些命令。

### Master 主机

```bash
pixi run upload-master
```

等价的 PlatformIO 原生命令：

```bash
platformio run -e master -t upload
```

### Slave 01 从机

```bash
pixi run upload-slave-01
```

等价的 PlatformIO 原生命令：

```bash
platformio run -e slave-01 -t upload
```

### Slave 02 从机

```bash
pixi run upload-slave-02
```

等价的 PlatformIO 原生命令：

```bash
platformio run -e slave-02 -t upload
```

### Motor 电机节点

```bash
pixi run upload-motor
```

等价的 PlatformIO 原生命令：

```bash
platformio run -e motor -t upload
```

## 只编译不烧录

如果只想先检查固件能否编译，通过下面命令构建：

```bash
pixi run build-master
pixi run build-slave-01
pixi run build-slave-02
pixi run build-motor
```

对应的 PlatformIO 原生命令：

```bash
platformio run -e master
platformio run -e slave-01
platformio run -e slave-02
platformio run -e motor
```

## 烧录注意事项

- `master` 和 `motor` 当前默认使用 `/dev/cu.usbmodem1101`。
- `slave-01` 和 `slave-02` 当前默认使用 `/dev/cu.usbmodem101`。
- 如果同一时间只连接一块 ESP32-S3，按上面的命令直接烧录即可。
- 如果连接多块板后端口号发生变化，需要先修改 `platformio.ini` 中对应环境的 `upload_port`，或者临时在命令中覆盖端口。

临时覆盖端口示例：

```bash
platformio run -e slave-01 -t upload --upload-port /dev/cu.usbmodem101
platformio run -e slave-02 -t upload --upload-port /dev/cu.usbmodem102
```

## 烧录顺序建议

采集链路建议按下面顺序烧录和上电测试：

```bash
pixi run upload-master
pixi run upload-slave-01
pixi run upload-slave-02
```

电机节点是独立 BLE 固件，可以单独烧录：

```bash
pixi run upload-motor
```

## 上位机运行

采集链路上位机：

```bash
pixi run host
```

查看可用串口：

```bash
pixi run host-list-ports
```

电机 BLE 控制上位机：

```bash
pixi run motor-host
```
