# 通信协议说明

这份文档说明当前固件分层后的通信协议。所有固件侧协议常量、CRC、大小端编解码和包构造函数集中在 `lib/capture_common`，Master、Slave 和上位机必须以这里的 wire format 为准。

## 1. 基本约定

- 字节序：小端。
- 校验：CRC16-CCITT，初值 `0xFFFF`，多字节 CRC 也按小端放入帧尾。
- IMU 采样率：1 kHz。
- IMU batch 大小：20 个样本。
- ESP-NOW 信道：`1`。
- BLE 只用于电机低频控制，不承载 IMU 高速采集数据。

核心 source：

| source | 含义 |
| --- | --- |
| `0x00` | Master 本地 ADS1298 EMG |
| `0x01` | `slave-01` Gyro |
| `0x02` | `slave-02` Accel |
| `0x7E` | Master 诊断帧 |

核心状态：

| state | 含义 |
| --- | --- |
| `0` | `STATE_IDLE`，待机，不采集高速数据 |
| `1` | `STATE_SYNC`，高频 Beacon，同步拟合 |
| `2` | `STATE_STREAM_PENDING`，等待统一采集起点 |
| `3` | `STATE_STREAM`，正式采集 |

## 2. ESP-NOW Beacon

方向：`Master -> Slave`，广播。

用途：提供 Master 时间基准，让 Slave 把本地 `micros()` 换算到 Master 时间轴。

```text
type:uint8 | beaconSeq:uint16 | state:uint8 | masterTimeUs:uint32
```

字段：

| 字段 | 值/类型 | 说明 |
| --- | --- | --- |
| `type` | `0x10` | Beacon 类型 |
| `beaconSeq` | `uint16_t` | Beacon 序号 |
| `state` | `uint8_t` | Master 当前 FSM 状态 |
| `masterTimeUs` | `uint32_t` | Master 发送 Beacon 时刻的 `micros()` |

总长度：8 字节。

SYNC 阶段 Slave 不再使用 EMA offset，而是收集多组：

```text
localRecvUs, masterTimeUs
```

并拟合：

```text
masterTimeUs = slope * localRecvUs + intercept
```

STREAM 阶段 IMU 样本时间戳使用冻结后的 `slope/intercept` 计算，不再用 `(oldOffset*7 + measuredOffset)/8`。

## 2.1 ESP-NOW Command

方向：`Master -> Slave`，广播。

用途：Master 将 PC 文本命令转成可靠的状态切换命令，连续广播多次。

```text
type:uint8 | commandSeq:uint16 | targetState:uint8 | effectiveMasterTimeUs:uint32 | crc16:uint16
```

总长度：10 字节。

命令语义：

| PC 文本命令 | targetState | 说明 |
| --- | --- | --- |
| `START_SYNC` | `STATE_SYNC` | 进入同步拟合阶段 |
| `START_STREAM` | `STATE_STREAM_PENDING` | 携带未来统一启动时间，默认 `now + 500ms` |
| `STOP` | `STATE_IDLE` | 停止采集并清空本轮缓存 |

## 2.2 ESP-NOW State ACK

方向：`Slave -> Master`，单播。

```text
type:uint8 | source:uint8 | state:uint8 | commandSeq:uint16 | effectiveMasterTimeUs:uint32 | crc16:uint16
```

总长度：11 字节。Master 会把它转成 PC 的 `PC_MSG_STATE_EVENT`。

## 2.3 ESP-NOW Sync Diag

方向：`Slave -> Master`，单播。

SYNC 阶段 Slave 每 200ms 上报一次线性拟合质量：

```text
type:uint8 | source:uint8 | state:uint8 | beaconSeq:uint16 |
offsetUs:int32 | driftPpm:int32 | residualUs:int32 | beaconCount:uint16 | crc16:uint16
```

总长度：21 字节。Master 会把它转成 PC 的 `PC_MSG_SYNC_DIAG`。

## 3. ESP-NOW IMU Batch

方向：`Slave -> Master`，单播。

用途：Slave 把 20 个 IMU 样本打成一个 batch 发给 Master。无线短时不稳定时，Slave 会保留 batch 并等待 ACK 或重传。

```text
type | source | batchSeq | sampleStartSeq | count | flags | baseTimeUs | sample[20] | crc16
```

字段：

| 字段 | 类型 | 长度 | 说明 |
| --- | --- | --- | --- |
| `type` | `uint8_t` | 1 | 固定 `0x20` |
| `source` | `uint8_t` | 1 | `0x01` Gyro，`0x02` Accel |
| `batchSeq` | `uint16_t` | 2 | batch 序号，用于 ACK、去重、重传 |
| `sampleStartSeq` | `uint32_t` | 4 | 本 batch 第一个样本序号 |
| `count` | `uint8_t` | 1 | 有效样本数，正常为 `20` |
| `flags` | `uint8_t` | 1 | 时间同步/重传/链路故障标志 |
| `baseTimeUs` | `uint32_t` | 4 | 本 batch 第一个样本的 Master 时间戳 |
| `sample[20]` | 20 * 8 | 160 | 相对时间和三轴数据 |
| `crc16` | `uint16_t` | 2 | 前面所有字节的 CRC |

总长度：

```text
14 + 20 * 8 + 2 = 176 bytes
```

单个 sample：

```text
dtUs:uint16 | x:int16 | y:int16 | z:int16
```

其中：

- `dtUs = timestampUs - baseTimeUs`
- Master 解码后还原为 `timestampUs = baseTimeUs + dtUs`

`flags`：

| bit | 名称 | 含义 |
| --- | --- | --- |
| `0x01` | `IMU_FLAG_RETRANSMIT` | 该 batch 是重传包 |
| `0x02` | `IMU_FLAG_TIME_SYNCED` | 样本时间戳已换算到 Master 时间轴 |
| `0x04` | `IMU_FLAG_LINK_FAULT` | 之前发生 ring buffer 溢出、同步中断或采样失败 |

## 4. ESP-NOW ACK

方向：`Master -> Slave`，单播。

用途：Master 确认已经收到的 batch，让 Slave 释放本地 ring buffer。ACK 丢失时 Slave 会重复发送，Master 负责去重。

```text
type | source | ackBaseSeq | ackSampleSeq | recvBitmap | masterTimeUs | crc16
```

字段：

| 字段 | 类型 | 长度 | 说明 |
| --- | --- | --- | --- |
| `type` | `uint8_t` | 1 | 固定 `0x11` |
| `source` | `uint8_t` | 1 | 被 ACK 的 Slave source |
| `ackBaseSeq` | `uint16_t` | 2 | 已连续确认到的最大 batch 序号 |
| `ackSampleSeq` | `uint32_t` | 4 | 与 `ackBaseSeq` 对应的最后样本序号 |
| `recvBitmap` | `uint32_t` | 4 | 后续 32 个 batch 的接收位图 |
| `masterTimeUs` | `uint32_t` | 4 | Master 发送 ACK 时刻 |
| `crc16` | `uint16_t` | 2 | 前面所有字节的 CRC |

总长度：18 字节。

Master 去重策略：

- 新 batch：转发给 PC，并发送 ACK。
- 重复 batch：只发送 ACK，不重复转发给 PC。
- CRC 错误：丢弃并计数。
- 接收窗口跳太远：统计 missing，表示实验段可能无效。

## 5. Master 到 PC 的 USB CDC 协议

方向：`Master -> tools/imu_scope.py`。

用途：把 EMG、Gyro、Accel、Diag 统一成一个二进制串口流。

CDC 批量帧：

```text
AA 55 | type | count | sequence | sample[0..count-1] | crc16
```

字段：

| 字段 | 类型 | 长度 | 说明 |
| --- | --- | --- | --- |
| `AA 55` | bytes | 2 | 帧头 |
| `type` | `uint8_t` | 1 | `0x30` 为采样 batch，`0x31` 为同步诊断，`0x32` 为状态事件 |
| `count` | `uint8_t` | 1 | 样本数量，`1..16` |
| `sequence` | `uint8_t` | 1 | CDC 帧序号，循环递增 |
| `sample[]` | `count * 24` | 变长 | `PcSample` 数组 |
| `crc16` | `uint16_t` | 2 | 整帧 CRC |

单个 `PcSample`：

```text
source:uint8
timestamp_us:uint32
sample_seq:uint32
batch_seq:uint16
flags:uint8
x:int32
y:int32
z:int32
```

总长度：24 字节。

说明：

- EMG 使用 Master 本地 `sample_seq`，`batch_seq=0`，`flags=0`。
- IMU 保留 Slave 的 `sample_seq`、`batch_seq` 和 `flags`。
- Diag 复用 `x/y/z`：
  - `x = forwardedPerSec`
  - `y = errorsPerSec`
  - `z = onlineSlaveCount`
- Master 缓存最近 64 个采样 batch 帧。PC 检测到 `sequence` 跳号后，可发送文本命令 `RESEND <sequence>` 请求重发对应采样帧；重发只覆盖 `type=0x30` 的采样 batch，不覆盖同步诊断和状态事件。

## 6. 上位机解析

`tools/imu_scope.py` 使用如下结构解析单样本：

```python
struct.Struct("<BIIHBiii")
```

CSV 输出字段包含：

```text
host_rx_time, source, source_id, timestamp_us, sample_seq, batch_seq, rx_flags, x, y, z, vector_norm
```

录制时会额外生成同步诊断 CSV：

```text
sync_diag_YYYYMMDD_HHMMSS.csv
```

字段：

```text
host_rx_time, source, source_id, state, beacon_seq, offset_us, drift_ppm, residual_us, beacon_count
```

做膝跳反射潜伏期分析前，建议先对每个 IMU source 检查：

```matlab
all(diff(sample_seq) == 1)
```

如果 `sample_seq` 不连续，或 `rx_flags` 出现链路故障标记 `0x04`，该实验段应标记为无效或重采。

## 7. 电机 BLE 协议

方向：`tools/motor_controller.py <-> motor node`。

BLE 设备名：

```text
Neuro_Hammer_BLE
```

Nordic UART Service：

| UUID | 方向 | 作用 |
| --- | --- | --- |
| `6E400001-B5A3-F393-E0A9-E50E24DCCA9E` | Service | BLE 串口服务 |
| `6E400002-B5A3-F393-E0A9-E50E24DCCA9E` | PC -> ESP32 | RX，写命令 |
| `6E400003-B5A3-F393-E0A9-E50E24DCCA9E` | ESP32 -> PC | TX，notify ACK/ERR |

命令均为 UTF-8 文本，以换行结尾：

| 命令 | 作用 |
| --- | --- |
| `PING` | 返回 `ACK: PONG` |
| `STRIKE_1` 到 `STRIKE_5` | 执行五档叩击 |
| `MOTOR_FWD` | 固定 PWM 正转 |
| `MOTOR_REV` | 固定 PWM 反转 |
| `STOP` | 立即停止 |
