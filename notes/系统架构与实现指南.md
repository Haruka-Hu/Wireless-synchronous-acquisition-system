# 系统架构与实现指南

这份文档用于从工程视角理解当前分层后的无线采集与电机控制系统。现在固件已经从“每个节点一个很大的 `main.cpp`”重构为嵌入式常见的四层结构：板级入口、节点应用层、驱动层、公共协议层。

## 1. 系统总体目标

工程包含两条相对独立的链路：

1. 多节点同步采集链路
   - Master 采集本地 ADS1298 肌电数据。
   - `slave-01` 采集小腿 IMU 陀螺仪数据。
   - `slave-02` 采集锤头 IMU 加速度数据。
   - Slave 通过 ESP-NOW 可靠 batch 链路发送给 Master。
   - Master 统一把 EMG、Gyro、Accel、诊断帧打包成 USB CDC 二进制流发给 `tools/imu_scope.py`。

2. 电机叩击控制链路
   - `motor` 节点单独烧录在一块 ESP32-S3 上。
   - `tools/motor_controller.py` 通过 BLE 发送五档叩击、正反转、急停命令。
   - 电机节点本地控制 BTS7960，不参与 ESP-NOW 采集链路。

```text
采集链路:
slave-01 Gyro --ESP-NOW--\
                          -> master --USB CDC-- tools/imu_scope.py
slave-02 Accel --ESP-NOW-/
master ADS1298 -----------/

控制链路:
tools/motor_controller.py --BLE-- motor node --PWM-- BTS7960 --Motor
```

职责划分：

| 路径 | 职责 |
| --- | --- |
| `src/*/main.cpp` | 只放板级配置和 App 启动入口。 |
| `lib/capture_common` | CRC、大小端编解码、ESP-NOW/ACK/USB CDC wire format。 |
| `lib/ads1298_driver` | ADS1298 SPI、复位、寄存器配置、CH6/CH7/CH8 读取。 |
| `lib/mpu6500_driver` | MPU6500 SPI、寄存器配置、Gyro/Accel 读取。 |
| `lib/bts7960_motor` | BTS7960 使能引脚、LEDC PWM、正反转和停止。 |
| `lib/master_node` | MasterApp：Beacon、ACK、去重、ADS 任务、CDC 输出。 |
| `lib/imu_slave_node` | ImuSlaveApp：时间同步、batch ring、错峰发送、重传。 |
| `lib/motor_node` | MotorApp：BLE NUS、命令解析、电机叩击状态机。 |

## 3. 多固件构建方式

工程继续使用 PlatformIO 的多环境结构：

```ini
[env:master]
build_src_filter =
    -<*>
    +<master/>
```

每个环境只编译自己的 `src/<env>/main.cpp`，但会按依赖自动编译 `lib/` 中被 include 的公共库。这样入口文件彼此隔离，公共逻辑又能复用。

常用命令：

```bash
pixi run build-master
pixi run build-slave-01
pixi run build-slave-02
pixi run build-motor

pixi run upload-master
pixi run upload-slave-01
pixi run upload-slave-02
pixi run upload-motor

pixi run host
pixi run motor-host
```

## 4. 固件分层设计

### 4.1 板级入口层

`src/master/main.cpp` 配置 ADS1298 引脚并启动 `MasterApp`。

`src/slave-01/main.cpp` 配置：

- `source = 0x01`
- `SensorMode::Gyro`
- 发送 slot `+2 ms`
- MPU6500 陀螺仪节点量程参数

`src/slave-02/main.cpp` 配置：

- `source = 0x02`
- `SensorMode::Accel`
- 发送 slot `+10 ms`
- MPU6500 加速度节点量程参数

`src/motor/main.cpp` 配置：

- BLE 名称和 Nordic UART Service UUID
- BTS7960 引脚：`RPWM=41`、`LPWM=42`、`REN=39`、`LEN=40`
- 五档叩击参数

### 4.2 节点应用层

`MasterApp` 负责 Master 业务调度：

- 初始化 ADS1298、ESP-NOW、USB CDC；
- 创建串口发送任务、无线任务、ADS 采样任务；
- 广播 Beacon；
- 接收和解码 Slave batch；
- 对 batch 去重并发送 ACK；
- 将 EMG/IMU/Diag 统一输出到 PC。

`ImuSlaveApp` 负责 Slave 业务调度：

- 初始化 MPU6500 和 ESP-NOW；
- 接收 Beacon 并维护 Master 时间偏移；
- 1 kHz 采样；
- 20 样本组 batch；
- 写入 256 batch 本地 ring buffer；
- 按 20 ms superframe 错峰发送；
- ACK 后释放 ring slot，ACK 丢失时重传。

`MotorApp` 负责电机节点业务调度：

- 初始化 BLE Nordic UART Service；
- 解析 `PING`、`STRIKE_1..5`、`MOTOR_FWD`、`MOTOR_REV`、`STOP`；
- 调用 `Bts7960Motor` 输出 PWM；
- 使用非阻塞状态机执行叩击动作，保证 `STOP` 可及时打断。

### 4.3 驱动层

驱动层不创建任务，也不理解业务协议。

- `Ads1298Driver` 只知道 ADS1298 的 SPI、GPIO、寄存器和通道读取。
- `Mpu6500Driver` 只知道 MPU6500 的 SPI、寄存器和三轴读取。
- `Bts7960Motor` 只知道 BTS7960 的使能脚、PWM 通道和正反转输出。

这种边界的好处是：以后换引脚、换量程、换电机驱动时，不需要改无线协议和任务调度代码。

### 4.4 公共协议层

`capture_common` 是采集链路 wire format 的唯一来源，包含：

- CRC16-CCITT；
- 小端整数 encode/decode；
- `BeaconPacket`；
- `ImuBatchWirePacket`；
- `ImuAckPacket` 的构造/解析；
- `PcSample` 和 USB CDC 批量帧构造。

Master 和 Slave 共享这份协议定义，避免两边各自复制常量后出现不一致。

## 5. ESP-NOW 可靠采集链路

当前采集链路继续使用 ESP-NOW，不改用 BLE。BLE 只用于低频电机控制。

Slave 端不再让采样任务直接等待无线发送，而是：

```text
1 kHz 采样 -> 20 样本 batch -> 256 batch ring buffer -> 无线任务错峰发送/重传
```

核心参数：

| 参数 | 值 |
| --- | --- |
| IMU 采样率 | 1 kHz |
| batch 大小 | 20 样本 |
| 单个 batch 时间 | 20 ms |
| ring buffer | 256 batch，约 5.12 s |
| superframe | 20 ms |
| slave-01 slot | +2 ms |
| slave-02 slot | +10 ms |
| 每个 slot 最多发送 | 3 个 batch |

无线 batch 格式：

```text
type | source | batchSeq | sampleStartSeq | count | flags | baseTimeUs | sample[20] | crc16
```

单个 sample 使用相对时间：

```text
dtUs:uint16 | x:int16 | y:int16 | z:int16
```

ACK 格式：

```text
type=0x11 | source | ackBaseSeq | ackSampleSeq | recvBitmap | masterTimeUs | crc16
```

Master 对重复 batch 的处理原则：

- 第一次收到：转发给 PC，并发送 ACK。
- 重复收到：只发送 ACK，不重复写入 PC。
- CRC 错误：丢弃并计数。
- 接收窗口跳太远：统计 missing，表示该实验段可能需要重采。

## 6. Master 到 PC 的 USB CDC 协议

USB CDC 帧：

```text
AA 55 | type=0x30 | count | sequence | sample[0..count-1] | crc16
```

单个 `PcSample`：

```text
source:uint8
timestamp_us:uint32
sample_seq:uint32
batch_seq:uint16
flags:uint8
x:int32
y:int32
z:int32
```

CSV 保留原来的常用字段，同时新增：

- `sample_seq`
- `batch_seq`
- `rx_flags`

潜伏期分析前建议先检查：

```matlab
all(diff(sample_seq) == 1)
```

只有加速度和角速度的 `sample_seq` 与 timestamp interval 都连续时，再计算膝跳反射潜伏期。

## 7. 电机控制链路

电机节点使用 BLE GATT，不使用经典蓝牙 SPP。原因是当前目标板是 ESP32-S3，更适合 BLE。

BLE 设备名：

```text
Neuro_Hammer_BLE
```

Nordic UART Service UUID：

| UUID | 方向 | 作用 |
| --- | --- | --- |
| `6E400001-B5A3-F393-E0A9-E50E24DCCA9E` | Service | BLE 串口服务 |
| `6E400002-B5A3-F393-E0A9-E50E24DCCA9E` | PC -> ESP32 | RX，写命令 |
| `6E400003-B5A3-F393-E0A9-E50E24DCCA9E` | ESP32 -> PC | TX，notify ACK/ERR |

命令：

| 命令 | 作用 |
| --- | --- |
| `PING` | 返回 `ACK: PONG` |
| `STRIKE_1` 到 `STRIKE_5` | 执行对应档位叩击 |
| `MOTOR_FWD` | 固定 PWM 正转 |
| `MOTOR_REV` | 固定 PWM 反转 |
| `STOP` | 立即停止 |

五档参数：

| 档位 | 回撤 PWM | 回撤 ms | 敲击 PWM | 敲击 ms |
| --- | --- | --- | --- | --- |
| 1 | 300 | 200 | 400 | 150 |
| 2 | 400 | 200 | 600 | 120 |
| 3 | 400 | 200 | 800 | 100 |
| 4 | 500 | 200 | 950 | 80 |
| 5 | 500 | 200 | 1023 | 70 |

## 8. 修改代码时的建议

- 改协议字段：优先改 `lib/capture_common`，再同步 Python 上位机。
- 改 IMU 寄存器或量程：改 `lib/mpu6500_driver` 或 `src/slave-*` 中的配置。
- 改 ADS1298 初始化：改 `lib/ads1298_driver`。
- 改无线可靠性策略：改 `lib/imu_slave_node` 和 `lib/master_node`。
- 改电机动作参数：优先改 `src/motor/main.cpp` 中的 `STRIKE_LUT`。
- 改 BLE 命令解析或状态机：改 `lib/motor_node`。
