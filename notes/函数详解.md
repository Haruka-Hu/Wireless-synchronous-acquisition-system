{ 0x02, 10000, Accel, {{10,11,13,12}, 0x10, 0x18}, 80, 2000000, 32 }

```

## 2. Master 初始化

### 2.1 `MasterApp(const Config &config)` — 构造函数

**文件**: `lib/master_node/master_node.cpp`

保存板级配置（ADS1298 引脚、串口波特率），并用 `config.adsPins` 构造 `Ads1298Driver` 实例。构造函数不做硬件操作。

---

### 2.2 `MasterApp::begin()` — 初始化入口

**文件**: `lib/master_node/master_node.cpp`

按顺序执行以下步骤：

1. `g_activeApp = this` — 设置全局实例指针，供 ISR 回调使用。
2. `Serial.setTxBufferSize(16384)` + `Serial.begin(2M)` — 初始化 USB CDC，TX 缓冲 16KB，写超时 1000ms。
3. `ads_.begin(onAdsDrdyStatic)` — 初始化 ADS1298 并绑定 DRDY 中断回调（见 §7.1）。
4. `xQueueCreate(512, SlaveRxItem)` — 创建 `slaveRxQueue_`，存放 ESP-NOW 收到的原始 batch（容量 512）。
5. `xQueueCreate(4096, PcSample)` — 创建 `serialQueue_`，存放所有待发给 PC 的统一样本（容量 4096 ≈ 1s 缓冲）。
6. `xQueueCreate(16, PcSerialTxFrame)` — 创建 `serialFrameQueue_`，存放状态、同步诊断和 USB 重发帧。
7. `xSemaphoreCreateMutex()` — 创建串口写和重发缓存互斥锁。
8. `initEspNow()` — 初始化 ESP-NOW（见 §2.3）。
9. 创建 4 个 FreeRTOS 任务（见 §3.1–§3.4）。

**关键参数**:
| 项目 | 值 |
|---|---|
| USB CDC TX buffer | 16384 字节 |
| Serial.setTxTimeoutMs | 1000ms |
| slaveRxQueue_ 容量 | 512 × ~193 字节 |
| serialQueue_ 容量 | 4096 × 24 字节 |
| serialFrameQueue_ 容量 | 16 × PC_SERIAL_MAX_PACKET_SIZE |

---

### 2.3 `MasterApp::initEspNow()` — ESP-NOW 初始化

**文件**: `lib/master_node/master_node.cpp`

1. `WiFi.mode(WIFI_STA)` — 设为 Station 模式。
2. `WiFi.disconnect()` + `WiFi.setSleep(false)` — 断开 WiFi 连接，禁止省电。
3. `esp_wifi_set_channel(1, WIFI_SECOND_CHAN_NONE)` — 锁定信道 1。
4. `esp_now_init()` — 初始化 ESP-NOW 协议栈。
5. `esp_now_add_peer(broadcastMac, ...)` — 添加广播 peer（FF:FF:FF:FF:FF:FF），用于发送 Beacon 和命令。
6. `esp_now_register_recv_cb(onEspNowRecvStatic)` — 注册接收回调。

**返回值**: 任一步骤失败返回 false。

---

### 2.4 `MasterApp::ensureEspNowPeer(mac)` — 确保 peer 存在

**文件**: `lib/master_node/master_node.cpp`

在发送单播 ACK 前调用。如果该 MAC 不在 peer 列表中，动态添加。防止因 peer 不存在导致 `esp_now_send` 失败。

**参数**: `mac[6]` — Slave 的 MAC 地址。
**返回值**: peer 存在或添加成功返回 true。

## 3. Master 运行时

Master 的 4 个 FreeRTOS 任务及其产生的中断回调：

| 任务/回调 | Core | 优先级 | 触发方式 |
|---|---|---|---|
| ADS1298 DRDY ISR | — | ISR | 硬件中断（每 500μs） |
| ESP-NOW Recv Callback | — | WiFi 任务上下文 | ESP-NOW 收到数据 |
| `sensorTask` | 1 | 5 | DRDY 通知唤醒 |
| `wirelessTask` | 0 | 7 | 2ms 定时循环 |
| `serialTxTask` | 0 | 6 | serialQueue 有数据时唤醒 |
| `serialCommandTask` | 0 | 3 | 5ms 定时轮询 Serial |

---

### 3.1 ISR 回调

#### `onAdsDrdyStatic()` → `onAdsDrdyFromIsr()`

**文件**: `lib/master_node/master_node.cpp`

ADS1298 DRDY 引脚下降沿触发（每 500μs 一次）。在 ISR 中**不读 SPI**，只做：

1. 调用 `vTaskNotifyGiveFromISR(sensorTaskHandle_)` 唤醒 `sensorTask`。
2. 如有更高优先级任务被唤醒，执行 `portYIELD_FROM_ISR()` 立即切换。

---

#### `onEspNowRecvStatic(mac, data, len)` → `onEspNowRecv(mac, data, len)`

**文件**: `lib/master_node/master_node.cpp`

ESP-NOW 接收回调。在 WiFi 任务上下文中运行，**不做解码**，只做：

1. 检查 `data` 长度 ≤ `IMU_BATCH_WIRE_SIZE`（176 字节）。
2. 检查 `data[0]` 是否为有效类型（`MSG_TYPE_IMU_BATCH` / `MSG_TYPE_SYNC_DIAG` / `MSG_TYPE_STATE_ACK`）。
3. 把 MAC + 原始字节拷贝到 `SlaveRxItem` 结构体。
4. `xQueueSend(slaveRxQueue_, &item, 0)` 写入队列。队列满则递增 `slaveQueueDrops_`。

**注意**: 这里用 0 超时，队列满直接丢弃，不在回调里阻塞。

---

### 3.2 `sensorTask()` — EMG 采样任务

**文件**: `lib/master_node/master_node.cpp`

启动时获取 `TaskHandle` 供 DRDY ISR 使用。

**循环逻辑**:

```
loop:
  ulTaskNotifyTake(pdTRUE, portMAX_DELAY)   // 等待 DRDY 通知
  ↓
  timestampUs = micros()                     // 记录采样时刻
  ↓
  if state == STATE_STREAM:                  // 只在 STREAM 态才读 ADC
    ads_.readChannels(ch6, ch7, ch8)         // SPI 读 CH6/CH7/CH8（§7.1）
    ↓
    writeSerialSample(SOURCE_EMG,            // 封装为 PcSample 入队
                      timestampUs,
                      emgSampleSeq_++,        // EMG 独立 sample 序号
                      0, 0,                   // batchSeq/flags 对 EMG 无意义
                      ch6, ch7, ch8)
```

**关键细节**:
- `emgSampleSeq_` 在 `resetStreamingState()` 中归零。
- `readChannels` 返回 false 时（当前代码永远不会），样本静默丢弃。
- 阻塞在 `ulTaskNotifyTake` 上，DRDY 停止则任务永久挂起。

---

### 3.3 `wirelessTask()` — 无线处理任务

**文件**: `lib/master_node/master_node.cpp`

这是 Master 最复杂的任务，负责 Beacon 广播、IMU 解码、ACK、去重、诊断。每 2ms 循环一次。

**循环逻辑（简化）**:

```
loop:
  nowMs = millis()
  state = currentState()

  // 1. 广播 Beacon
  if nowMs - lastBeaconMs >= beaconIntervalMs(state):
    broadcastBeacon(micros())                // §3.3.1

  // 2. STREAM_PENDING → STREAM 自动转换
  if state == STREAM_PENDING and micros() >= pendingStreamStartUs_:
    resetStreamingState()                    // §3.3.8
    state_ = STATE_STREAM
    writePcStateEvent(STREAM)               // 通知 PC

  // 3. 处理所有待处理的 ESP-NOW 接收数据
  while xQueueReceive(slaveRxQueue_, &rxItem, 0):
    if item 是 STATE_ACK:   writePcStateEvent → trackSlave
    if item 是 SYNC_DIAG:   writePcSyncDiag  → trackSlave
    if item 是 IMU_BATCH and state == STREAM:
      decodeImuBatchPacket(...)              // §6.x — CRC 校验 + 解码
      ↓
      markBatchReceived(...)                 // §3.3.4 — 去重检查
      ↓
      trackSlave(...)                        // §3.3.5 — 更新在线状态
      ↓
      if 不是重复包:
        for each decoded sample:
          writeSerialSample(...)             // 写入 serialQueue_

  // 4. 发送待发的 ACK
  sendPendingAcks(micros())                  // §3.3.6

  // 5. 每秒诊断统计
  if nowMs - lastDiagMs >= 1000:
    计算各项 delta（forwarded/errors/linkEvents/queueDrops...）
    writeDiagFrame(...)                      // 写入 serialQueue_

  vTaskDelay(2ms)
```

---

#### 3.3.1 `broadcastBeacon(nowUs)` — 广播时间信标

**文件**: `lib/master_node/master_node.cpp`

构造 `BeaconPacket{type=0x10, beaconSeq++, state, nowUs}`，通过 `esp_now_send(broadcastMac)` 广播。

**频率**:
| 状态 | 间隔 |
|---|---|
| IDLE | 2000ms |
| SYNC | 20ms |
| STREAM_PENDING | 20ms |
| STREAM | 200ms |

---

#### 3.3.2 `beaconIntervalMs(state)` — 查询 Beacon 间隔

**文件**: `lib/master_node/master_node.cpp`

返回当前状态对应的 Beacon 广播间隔。

---

#### 3.3.3 `currentState()` — 读取当前状态

**文件**: `lib/master_node/master_node.h`

直接返回 `volatile uint8_t state_`。被多个任务读取（包括 `sensorTask` 在 Core 1 上），volatile 保证跨核心可见。

---

#### 3.3.4 `markBatchReceived(state, batchSeq, sampleStartSeq, flags, outDuplicate)` — 去重和窗口管理

**文件**: `lib/master_node/master_node.cpp`

在转发 IMU 样本给 PC 之前，判断 batch 是否已收到过。

**算法**（32 位滑动窗口）:

```
distance = batchSeq - ackBaseSeq           // 到连续基准的距离

if distance == 0 or > 32768:               // 序列号回绕的重复
    标记为重复 → 不转发

if distance > 32:                          // 超出窗口，中间全部标记 missing
    missingPackets += distance - 1
    重新锚定窗口：ackBaseSeq = batchSeq - 1

mask = 1 << (distance - 1)
if recvBitmap & mask:                      // 该位置已有
    标记为重复 → 不转发

recvBitmap |= mask                         // 标记已收到
advanceAckWindow(state)                    // §3.3.4.1
```

**首次初始化**: 第一个 batch 将 `ackBaseSeq` 设为 `batchSeq - 1`，作为窗口起点。

---

#### 3.3.4.1 `advanceAckWindow(state)` — 推进 ACK 基准

**文件**: `lib/master_node/master_node.cpp`

当 `recvBitmap` 最低位为 1 时，说明 `ackBaseSeq + 1` 已收到：

```
while recvBitmap & 0x01:
    ackBaseSeq += 1
    ackSampleSeq += IMU_BATCH_SIZE
    recvBitmap >>= 1
```

这使得 `ackBaseSeq` 始终指向连续收到的最后一个 batch。

---

#### 3.3.5 `trackSlave(mac, source, nowUs, sampleCount)` — 更新在线状态

**文件**: `lib/master_node/master_node.cpp`

维护 `slaveTrackers_[]` 数组（最多 16 个）。记录每个已知 Slave 的：
- `lastSeenUs` — 最近收到数据的时间
- `totalSamples` — 累计样本数
- `source` — 节点编号

队列满时覆盖最久未见的记录。

---

#### 3.3.6 `sendPendingAcks(nowUs)` — 限速发送 ACK

**文件**: `lib/master_node/master_node.cpp`

遍历所有 `slaveRxStates_[]`：

```
for each SlaveRxState:
    if used && ackPending:
        if nowUs - lastAckSentUs >= ACK_MIN_INTERVAL_US (20ms):
            sendAck(state, nowUs)          // §3.3.6.1
```

**限速原因**: 防止 ACK 风暴打满 ESP-NOW 发送队列。

---

#### 3.3.6.1 `sendAck(state, nowUs)` — 发送单个 ACK

**文件**: `lib/master_node/master_node.cpp`

1. `ensureEspNowPeer(state.mac)` — 确保单播 peer 存在。
2. `capture::buildAckPacket(source, ackBaseSeq, ackSampleSeq, recvBitmap, nowUs, ack)` — 构造 18 字节 ACK 包。
3. `esp_now_send(state.mac, ack, 18)` — 单播发送。
4. 失败时递增 `ackSendFails_`，`ackPending` 保持 true 以便下次重试。

---

#### 3.3.7 `getSlaveRxState(mac, source)` — 获取/创建接收窗口

**文件**: `lib/master_node/master_node.cpp`

在 `slaveRxStates_[]` 数组中按 MAC + source 查找已有状态。不存在则使用空闲槽位或覆盖最旧的。

---

#### 3.3.8 `resetStreamingState()` — 重置流状态

**文件**: `lib/master_node/master_node.cpp`

在进入 IDLE、SYNC、STREAM_PENDING 状态时调用：
- `emgSampleSeq_ = 0`
- 所有 `slaveRxStates_[]` 清空
- `xQueueReset(serialQueue_)` 清空串口队列

---

#### 3.3.9 `countOnlineSlaves(nowUs)` — 统计在线 Slave 数

**文件**: `lib/master_node/master_node.cpp`

遍历 `slaveTrackers_[]`，统计 `nowUs - lastSeenUs < 3s` 的 Slave 数量。用于诊断帧。

---

### 3.4 `serialTxTask()` — 串口合批发送任务

**文件**: `lib/master_node/master_node.cpp`

`serialTxTask` 是 Master 唯一真正调用 `Serial.write()` 的任务。它优先发送 `serialFrameQueue_` 中的状态、同步诊断和 USB 重发帧；空闲时再从 `serialQueue_` 取样本并合批编码为 USB CDC 帧。

```
loop:
  if serialFrameQueue_ 有帧:
    writeSerialBytesAll(frame.bytes, frame.size)
    continue

  if serialQueue_ 有样本:
    batch[0] = sample, sampleCount = 1
    while sampleCount < 16:
      xQueueReceive(serialQueue_, &sample, 1ms)
    buildPcBatchPacket(batch, sampleCount, seq++, buffer)
    cachePcSerialFrame(seq, buffer, size)
    writeSerialBytesAll(buffer, size)
```

---

#### 3.4.1 `writeSerialBytesAll(data, size)` — 阻塞式完整写出

**文件**: `lib/master_node/master_node.cpp`

`Serial.write()` 可能短写（一次写不完所有字节），本函数循环补齐：

```
xSemaphoreTake(serialWriteMux_, portMAX_DELAY)           // 获取写锁
startMs = millis()
while writtenTotal < size:
    written = Serial.write(data + writtenTotal, size - writtenTotal)
    if written == 0:
        serialWriteShorts_++
        if millis() - startMs > 500ms:                   // 500ms 超时放弃
            serialWriteAborts_++
            break
        vTaskDelay(1ms)
    writtenTotal += written
xSemaphoreGive(serialWriteMux_)
```

---

#### 3.4.2 `writeSerialSample(source, timestampUs, sampleSeq, batchSeq, flags, x, y, z)` — 写入一条样本到串口队列

**文件**: `lib/master_node/master_node.cpp`

封装 `PcSample{source, timestampUs, sampleSeq, batchSeq, flags, x, y, z}`，`xQueueSend` 到 `serialQueue_`（0 超时，满则丢弃并递增 `serialQueueDrops_`）。

EMG、IMU、Diag 三种数据源的样本都通过这个函数入队。

---

### 3.5 `serialCommandTask()` — PC 命令处理任务

**文件**: `lib/master_node/master_node.cpp`

每 5ms 轮询 `Serial.available()`。读到换行符时调用 `handlePcCommand(line)`。

---

#### 3.5.1 `handlePcCommand(raw)` — 解析 PC 文本命令

**文件**: `lib/master_node/master_node.cpp`

| 命令 | 动作 |
|---|---|
| `PING` | 返回当前状态 |
| `START_SYNC` | `transitionTo(STATE_SYNC, 0)` |
| `START_STREAM` | `transitionTo(STATE_STREAM_PENDING, micros()+500000)` |
| `STOP` | `transitionTo(STATE_IDLE, 0)` |

---

#### 3.5.2 `transitionTo(newState, effectiveMasterTimeUs)` — 状态切换

**文件**: `lib/master_node/master_node.cpp`

1. 进入 IDLE/SYNC/STREAM_PENDING 时调用 `resetStreamingState()`。
2. `state_ = newState`。
3. `broadcastCommand(newState, effectiveMasterTimeUs)` — 通过 ESP-NOW 广播给所有 Slave。
4. `writePcStateEvent(event)` — 通知 PC 端状态变化。

---

#### 3.5.3 `broadcastCommand(targetState, effectiveMasterTimeUs)` — 广播状态命令

**文件**: `lib/master_node/master_node.cpp`

构造 `CommandPacket{++commandSeq_, targetState, effectiveMasterTimeUs}`，通过 `esp_now_send(broadcastMac)` 广播 **6 次**，每次间隔 8ms。6 次重复确保在丢包环境下 Slave 也能收到。

---

### 3.6 事件发送函数

#### 3.6.1 `writePcStateEvent(event)` — 发送状态事件到 PC

**文件**: `lib/master_node/master_node.cpp`

调用 `capture::buildPcStateEventPacket()` 构造二进制帧，放入 `serialFrameQueue_` 而非直接写串口。由 `serialTxTask` 统一写出，避免 `wirelessTask` 高优先级直接阻塞在 `Serial.write` 上。

---

#### 3.6.2 `writePcSyncDiag(diag)` — 转发同步诊断到 PC

**文件**: `lib/master_node/master_node.cpp`

同上，构造同步诊断帧后放入 `serialFrameQueue_`。

---

#### 3.6.3 `writeDiagFrame(timestampUs, forwardedPerSec, ...)` — 秒级诊断帧

**文件**: `lib/master_node/master_node.cpp`

把链路质量指标（转发量、错误量、在线 Slave 数、链路事件、队列丢弃、ACK 失败）封装为 `SOURCE_DIAG` 样本，写入 `serialQueue_`（与普通样本同一通道）。

**PcSample 字段复用**:

| PcSample 字段 | 实际含义 |
|---|---|
| `x` | 每秒转发样本数 |
| `y` | 每秒硬件错误数 |
| `z` | 在线 Slave 数 |
| `sampleSeq` | 链路事件数 |
| `batchSeq` | 队列丢弃数 |
| `flags` | ACK 发送失败数 |

## 4. Slave 初始化

### 4.2 `ImuSlaveApp::begin()` — 初始化入口

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

### 4.3 `ImuSlaveApp::initEspNow()` — ESP-NOW 初始化

3. `esp_wifi_set_max_tx_power(config.espnowTxPowerQdbm)` — 设置发射功率（32 qdBm = 8 dBm）。
4. `esp_now_init()`。
5. `esp_now_register_send_cb(onEspNowSentStatic)` — 注册发送完成回调。
6. `esp_now_register_recv_cb(onEspNowRecvStatic)` — 注册接收回调。

**注意**: Slave 初始化时不添加 peer，peer 在收到第一个 Beacon 后动态添加（获取 Master MAC）。

## 5. Slave 运行时

Slave 的 2 个 FreeRTOS 任务及其回调：

| 任务/回调 | Core | 优先级 | 触发方式 |
|---|---|---|---|
| ESP-NOW Send Callback | — | WiFi 任务上下文 | 发送完成 |
| ESP-NOW Recv Callback | — | WiFi 任务上下文 | 收到数据 |
| `sensorTask` | 1 | 5 | 1kHz 软件定时 |
| `wirelessTask` | 0 | 4 | 2ms 定时循环 |

---

### 5.1 回调

#### `onEspNowRecvStatic(mac, data, len)` → `onEspNowRecv(mac, data, len)`

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

分发三类下行包：

| data[0] | 长度 | 处理函数 |
|---|---|---|
| `MSG_TYPE_COMMAND (0x12)` | 10 字节 | `handleCommandPacket()` |
| `MSG_TYPE_IMU_ACK (0x11)` | 18 字节 | `handleAckPacket()` |
| 其他 | 9 字节 | `handleBeacon()` |

每次收到包时，记录 Master MAC（`hasMasterMac_ = true`）。

---

#### `onEspNowSentStatic(mac, status)` → `onEspNowSent(status)`

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

记录 `lastSendOk_ = (status == ESP_NOW_SEND_SUCCESS)`，并通过 `xSemaphoreGiveFromISR(sendDoneSem_)` 唤醒等待的 `wirelessTask`。

### 5.2 `handleBeacon(mac, data, len)` — 处理 Beacon

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

1. 校验长度 == `sizeof(BeaconPacket)`（9 字节，packed）。
2. 校验 `type == MSG_TYPE_BEACON (0x10)`。
3. 在 `offsetMux_` 保护下：
   - 处于 SYNC/STREAM_PENDING/STREAM 状态 → 调用 `recordBeaconLocked()`。
   - 保存 Master MAC。

---

#### 5.2.1 `recordBeaconLocked(localRecvUs, masterTimeUs, beaconSeq)` — 记录同步点

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

将 `{localRecvUs, masterTimeUs}` 写入 128 点的循环窗口 `syncPoints_[]`，覆盖最旧的点。更新 `lastBeaconSeq_`。

---

#### 5.2.2 `updateClockFitLocked()` — 更新线性时钟模型

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

两遍鲁棒最小二乘拟合：

**第一遍**: `fitClockModelLocked(nullptr, slope, intercept, residualRms)` — 对所有窗口点做普通最小二乘。

**第二遍**（需要 ≥ 8 个点）:
1. 计算每个点的残差绝对值。
2. 排序，取阈值 = min(第 80 百分位, 5000us)。
3. 过滤残差超过阈值的离群点（最多排除 20%）。
4. 用保留点重新拟合。

**结果**:
```
clockSlope_     = slope        // 时钟频率比 (≈1.0)
clockIntercept_ = intercept    // 时钟偏移
syncOffsetUs_   = nowMaster - nowLocal   // 当前偏移
syncDriftPpm_   = (slope - 1) × 1e6      // ppm 漂移
syncResidualUs_ = residualRms  // 拟合残差 RMS
clockModelValid_ = true
```

---

#### 5.2.3 `fitClockModelLocked(includeMask, slope, intercept, residualRms)` — 单次最小二乘

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

以窗口中最旧的点为基准，计算相对时间差，做普通最小二乘：

```
masterUs = slope × localUs + intercept
```

### 5.3 `handleCommandPacket(data, len)` — 处理状态命令

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

解码 `CommandPacket`，调用 `applyState(targetState, commandSeq, effectiveMasterTimeUs)`。

---

#### 5.3.1 `applyState(newState, commandSeq, effectiveMasterTimeUs)` — 状态切换

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

1. 去重：`commandSeq` 等于 `lastCommandSeq_` 则只回 ACK，不重复处理。
2. 进入 IDLE/SYNC/STREAM_PENDING → 调用 `resetRingLocked()` 清空 ring buffer。
3. 进入 SYNC → 调用 `resetSyncFitLocked()` 重置时钟拟合。
4. 设置 `state_ = newState`、`pendingStreamStartUs_`、`streamStarted_ = false`。
5. 调用 `sendStateAck()` 向 Master 确认状态变更。

---

#### 5.3.2 `sendStateAck(commandSeq, effectiveMasterTimeUs)` — 发送状态 ACK

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

构造 `StateAckPacket{source, state, commandSeq, effectiveMasterTimeUs}`，通过 `esp_now_send` 单播给 Master。如果 Master 不在 peer 列表中，先添加。

---

#### 5.3.3 `resetRingLocked()` — 清空 ring buffer

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

将所有 `batchRing_[]` 槽位置为 unused，重置 `nextBatchSeq_ = 0`、`oldestUnackedSeq_ = 0`、`nextSampleSeq_ = 0`，清零所有计数器。

---

#### 5.3.4 `resetSyncFitLocked()` — 重置时钟拟合

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

将 `clockSlope_` 重置为 1.0、`clockIntercept_` 重置为 0.0，清零同步窗口。

---

### 5.4 `handleAckPacket(data, len)` — 处理 ACK

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

1. `capture::decodeAckPacket(data, len, nodeSource, ackBaseSeq, recvBitmap)` — 校验 CRC 和 source。
2. 遍历 `batchRing_[]`：`ackCoversBatchSeq(slot.batchSeq, ackBaseSeq, recvBitmap)` 为 true 的槽位释放（`slot.used = false`）。
3. 递增 `ackPackets_` 和 `ackedBatches_`。
4. 调用 `advanceOldestUnackedLocked()`。

---

#### 5.4.1 `advanceOldestUnackedLocked()` — 推进最早未 ACK 指针

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

从 `oldestUnackedSeq_` 开始扫描 ring buffer，跳过空闲槽位和被覆盖的旧 batchSeq，直到找到第一个仍在使用且 batchSeq 匹配的槽位。确保 `takeSendCandidate()` 从正确的位置开始搜索。

### 5.5 `sensorTask()` — IMU 采样任务

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

1kHz 软件定时采样。每次迭代：

```
loop:
  if state != STREAM or !streamStarted_:            // 非流状态
    vTaskDelay(2ms)
    continue

  while micros() < nextSampleUs: taskYIELD()        // 忙等待到 1kHz 采样点
  nextSampleUs += 1000

  localTimeUs = micros()
  sample.timestampUs = localToSampleTimeUs(localUs) // §5.5.1 — 换算到 Master 时间
  readSensor(sample)                                 // §5.5.2 — 读 MPU6500
  batchSamples[batchCount++] = sample

  if batchCount >= 20:                               // 满 1 batch
    storeBatch(batchSamples, 20)                     // §5.5.3 — 写入 ring buffer
    batchCount = 0
```

---

#### 5.5.1 `localToSampleTimeUs(localUs)` — 换算到 Master 采样时间

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

优先使用**冻结的**采样时钟模型（`sampleClockSlope_` / `sampleClockIntercept_`）。该模型在 STREAM 开始时由 `freezeSampleClockLocked()` 冻结，确保流式传输期间样本时间戳不受后续 Beacon 更新的影响（避免时间轴跳变）。

若冻结模型无效，降级使用实时模型 `localToMasterTimeUs()`。

---

#### 5.5.2 `localToMasterTimeUs(localUs)` — 实时时钟转换

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

使用实时更新的 `clockSlope_` / `clockIntercept_`（每 200ms 更新一次）进行转换：
```
masterUs = clockSlope_ × localUs + clockIntercept_
```

用于 `wirelessTask` 的 TDMA 时隙计算。

---

#### 5.5.3 `readSensor(sample)` — 读取传感器数据

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

根据 `config_.sensorMode`：
- `Gyro` → `mpu_.readGyro(x, y, z)`
- `Accel` → `mpu_.readAccel(x, y, z)`

读取失败时调用 `noteDroppedSamples()`。

---

#### 5.5.4 `noteDroppedSamples(count)` — 记录丢样

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

当传感器读取失败或样本无法保存时，推进 `nextSampleSeq_` 跳过相应的 sample 序号，并设置 `linkFaultPending_ = true`。下一 batch 会带上 `IMU_FLAG_LINK_FAULT` 标记。

---

#### 5.5.5 `storeBatch(samples, count)` — 写入 ring buffer

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

1. 调用 `capture::makeImuBatchPacket(source, samples, count, batchSeq, sampleStartSeq)` 构造 176 字节 batch。
2. 在 `ringMux_` 保护下：
   - 取 `batchRing_[batchSeq % 256]` 槽位。
   - 若该槽位仍有未 ACK 的旧 batch → **覆盖**旧数据（优先保证实时性）并递增 `ringOverflows_`。
   - 写入新 batch 数据。
3. 递增 `nextBatchSeq_` 和 `nextSampleSeq_`。

---

### 5.6 `wirelessTask()` — 无线发送任务

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

核心逻辑按 20ms 超帧 + 错峰时隙调度发送。

```
loop:
  if 无 Master MAC: vTaskDelay(2ms); continue

  // 1. 确保 Master 在 peer 列表中
  if !esp_now_is_peer_exist(masterMac):
    esp_now_add_peer(masterMac)

  // 2. 同步诊断（每 200ms）
  if state 是 SYNC/STREAM_PENDING: sendSyncDiag()      // §5.6.1
  if state 是 STREAM:              updateClockFitLocked() // 静默更新

  // 3. STREAM_PENDING → STREAM 自动转换
  if state == STREAM_PENDING and localToMasterTimeUs(micros()) >= pendingStreamStartUs_:
    state_ = STATE_STREAM
    streamStarted_ = true
    freezeSampleClockLocked()                           // §5.6.2
    sendStateAck(...)

  // 4. 非 STREAM 状态：等待
  if state != STREAM: vTaskDelay(2ms); continue

  // 5. TDMA 时隙判断
  nowUs = micros()
  masterNowUs = localToMasterTimeUs(nowUs)              // 用实时模型
  slotIndex = masterNowUs / 20000
  phaseUs   = masterNowUs % 20000

  if slotIndex == lastSlotIndex:                        // 本超帧已发过
    vTaskDelay(1ms); continue
  if phaseUs < sendSlotOffsetUs:                        // 还没到时隙
    vTaskDelay(1ms); continue
  if phaseUs >= sendSlotOffsetUs + 5000:                // 时隙已过
    vTaskDelay(1ms); continue

  lastSlotIndex = slotIndex

  // 6. 发送（每次最多 3 个 batch）
  for sendCount = 0 to 2:
    if !takeSendCandidate(packet, micros()): break       // §5.6.3
    esp_now_send(masterMac, packet.bytes, 176)
    等待发送完成（8ms 超时）
```

---

#### 5.6.1 `sendSyncDiag()` — 发送同步诊断

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

调用 `updateClockFitLocked()` 更新拟合，然后构造 `SyncDiagPacket{source, state, beaconSeq, offsetUs, driftPpm, residualUs, beaconCount}`，通过 `esp_now_send` 发给 Master。

---

#### 5.6.2 `freezeSampleClockLocked()` — 冻结采样时钟

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

在 STREAM 启动瞬间，将当前的实时时钟模型复制为采样时钟模型：

```
sampleClockSlope_     = clockSlope_
sampleClockIntercept_ = clockIntercept_
sampleClockModelValid_ = clockModelValid_
```

之后所有 IMU 样本时间戳使用此冻结模型，不受后续 Beacon 微调影响。

---

#### 5.6.3 `takeSendCandidate(outPacket, nowUs)` — 获取发送候选

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

从 `oldestUnackedSeq_` 开始遍历 ring buffer，找第一个满足条件的 batch：

1. 槽位空闲或 batchSeq 不匹配 → 跳过。
2. batch 落后 `nextBatchSeq_` 超过 64 → **放弃**（太旧，Master 已判定 missing）。
3. 距上次发送不足 `RETRANSMIT_INTERVAL_US`（40ms）→ 跳过。
4. 满足条件 → 复制 batch，若 `sendAttempts > 0` 则设置 `IMU_FLAG_RETRANSMIT`，更新时间戳。

**返回值**: 找到候选项返回 true，否则 false。

---

#### 5.6.4 `loadMasterMac(outMac)` / `loadClockModel(slope, intercept)` / `loadSampleClockModel(slope, intercept)` — 线程安全读取

**文件**: `lib/imu_slave_node/imu_slave_node.cpp`

在 `offsetMux_` 临界区保护下读取共享状态。因为 `offsetMux_` 也保护 ESP-NOW 回调中对这些字段的写入。

### 7.1 ADS1298 驱动

```
